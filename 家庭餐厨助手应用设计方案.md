# 家庭餐厨助手应用设计方案

## 🍽️ 应用名称建议
- 简家厨
- 亲子餐盒
- 双厨记
- 宝宝餐大人餐

## 📋 产品概述

### 核心理念
一款为有婴幼儿的家庭设计的餐食规划应用，**同时提供大人和宝宝（1.5岁）两种餐食方案**，强调简单、快速、营养均衡。

### 目标用户
- 家有1.5岁左右宝宝的父母
- 工作繁忙但需兼顾全家饮食的上班族
- 新手父母，需要营养指导
- 想减少思考“今天吃什么”的时间

## 🎯 核心功能模块

### 模块一：智能推荐系统（MVP核心）
```
每日推荐 = 大人餐 + 宝宝餐 + 购物清单
```
- **配对逻辑**：每道大人餐配搭适配的宝宝版
- **筛选条件**：
  - 准备时间：<30分钟
  - 难度：初级
  - 食材：常见易购
  - 烹饪方式：蒸、煮、快炒为主

### 模块二：餐食详情页
```
【红烧鸡翅（大人版）】
├─ 准备时间：25分钟
├─ 所需食材：
│  - 鸡翅中 8个
│  - 生姜 3片
│  - 酱油 1勺
│  └─ ...
├─ 步骤：
│  1. 鸡翅洗净划刀（5分钟）
│  2. 冷水下锅焯水（5分钟）
│  3. 锅中少量油煎至金黄（5分钟）
│  4. 加入调料焖煮（10分钟）
└─ 小贴士：可提前腌制更入味

【鸡翅蔬菜泥（宝宝版）】
├─ 准备时间：15分钟（可同步制作）
├─ 食材调整：
│  - 取2个已焯水的鸡翅
│  - 额外添加：胡萝卜30g、西兰花2朵
├─ 分开制作步骤：
│  1. 鸡翅去骨取肉（用大人版已焯水的）
│  2. 蔬菜蒸熟（10分钟）
│  3. 混合打成泥（5分钟）
└─ 营养提示：提供优质蛋白质和维生素
```

### 模块三：智能购物清单

## 💡 技术实现建议

### 方案A：零代码快速原型（推荐先尝试）
- **使用平台**：Glide或Adalo
- **优点**：2天内可出可用的原型
- **数据管理**：Google Sheets作为数据库
- **适合**：验证需求，快速给朋友使用

### 方案B：混合开发（平衡速度与功能）
- **框架**：Flutter或React Native
- **后端**：Firebase（免费额度足够）
- **优点**：性能较好，功能扩展性强
- **时间**：2-3周完成MVP

### 方案C：原生开发（长期维护）
- **语言**：Kotlin（Android）或 Swift（iOS）
- **优点**：最佳性能，全功能
- **时间**：1个月以上

## 📊 初始菜谱库建议

### 配对原则：
1. **食材复用型**（如：大人吃鱼，宝宝吃鱼粥）
2. **工序同步型**（如：同时蒸煮，后分开调味）
3. **余料利用型**（如：大人餐多余食材做宝宝餐）

### 首批15对菜谱示例：

| 大人餐     | 宝宝餐     | 准备时间 | 核心技巧                 |
| ---------- | ---------- | -------- | ------------------------ |
| 番茄炒蛋   | 番茄蛋花粥 | 20分钟   | 先做宝宝粥底，再炒大人菜 |
| 清蒸鲈鱼   | 鲈鱼蔬菜泥 | 25分钟   | 一起蒸，宝宝的去刺压泥   |
| 肉末豆腐   | 豆腐肉末羹 | 15分钟   | 调味前分出一部分         |
| 南瓜焖饭   | 南瓜鸡肉泥 | 30分钟   | 电饭锅一锅出             |
| 蔬菜鸡肉卷 | 鸡肉蔬菜碎 | 25分钟   | 鸡肉先蒸熟，再做卷       |
