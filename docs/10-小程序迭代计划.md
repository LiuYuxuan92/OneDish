# OneDish 小程序迭代计划

> AI 可直接执行此计划。每个任务包含具体文件路径、当前状态、验收标准。

## 现状

- 4 个页面：login、home、recipe、plan
- MVP 骨架，核心功能未闭环
- 后端 API 已完备

---

## Phase 1：核心闭环（MVP 完善）

目标：让用户能用小程序完成"推荐→查看→下单"的基础闭环

### Task 1.1：完善首页今日推荐

**当前**：`pages/home/home.js` 仅拉取 `/recipes/daily`，无"一鱼两吃"逻辑

**目标**：显示推荐菜谱的成人版 + 宝宝版双版本

**文件**：`pages/home/home.js`, `pages/home/home.wxml`

**实现**：
1. 调用 `GET /recipes/daily` 获取推荐
2. 判断返回数据是否有 `babyVersion` 字段
3. 如果有，显示双版本卡片（成人版/宝宝版）
4. 点击切换版本显示

**验收**：首页能看到成人版/宝宝版切换

---

### Task 1.2：菜谱详情页改造

**当前**：`pages/recipe/recipe.js` 仅 46 行，只显示简单信息

**目标**：完整展示成人版 + 宝宝版菜谱详情

**文件**：`pages/recipe/recipe.js`, `pages/recipe/recipe.wxml`, `pages/recipe/recipe.wxss`

**实现**：
1. 调用 `GET /recipes/:id` 获取完整详情
2. 数据结构应包含：`title`, `adultVersion`, `babyVersion`, `ingredients`, `steps`, `babyAgeRange`
3. 页面顶部显示标题 + 版本切换 Tab
4. 成人版内容：食材、步骤、烹饪小贴士
5. 宝宝版内容：对应食材（无调味料）、简化步骤、月龄建议
6. 底部添加"加入购物清单"按钮

**验收**：
- 能查看完整菜谱
- 能切换成人/宝宝版本
- 能加入购物清单

---

### Task 1.3：购物清单功能闭环

**当前**：`pages/plan/plan.js` 183 行，有本地存储降级

**目标**：
1. 支持添加食材到清单
2. 支持从菜谱一键导入
3. 支持标记已买
4. 支持分享到微信群

**文件**：`pages/plan/plan.js`, `pages/plan/plan.wxml`, `pages/plan/plan.wxss`

**实现**：
1. 页面结构：
   - Tab1：当前清单（可勾选、删除）
   - Tab2：历史清单
2. 添加食材：底部输入框 + 确认按钮
3. 一键导入：从菜谱详情页传入食材列表
4. 勾选状态：`wx.setStorageSync` 本地持久化
5. 分享功能：`onShareAppMessage` 分享当前清单到微信群
6. 清单格式：{ id, name, quantity, unit, checked }

**验收**：
- 能添加、勾选、删除食材
- 能从菜谱详情导入
- 能分享到微信群

---

### Task 1.4：微信登录

**当前**：`pages/login/login.js` 116 行

**目标**：获取用户微信头像昵称，调后端接口换取 token

**文件**：`pages/login/login.js`, `pages/login/login.wxml`

**实现**：
1. 按钮点击调用 `wx.getUserProfile` 获取用户信息
2. 调用 `wx.login` 获取 code
3. 调后端 `POST /auth/wechat` 换取 token
4. token 存 `wx.setStorageSync('token', token)`
5. 登录成功后跳转首页

**验收**：登录后能获取 token，后续接口带 token

## Phase 2：增长与粘性

目标：让用户愿意回来、愿意分享

### Task 2.1：宝宝月龄设置

**目标**：让用户设置宝宝年龄，推荐更精准

**文件**：新建 `pages/profile/profile.js`（或放在 login 页）

**实现**：
1. 入口：首页右上角"设置"按钮
2. 页面：选择宝宝出生日期或月龄（0-36个月）
3. 存储：`wx.setStorageSync('babyAge', age)`
4. 首页推荐时带上 babyAge 参数

**验收**：能设置并保存宝宝月龄

---

### Task 2.2：搜索功能

**当前**：无搜索页面

**目标**：按食材、菜名搜索菜谱

**文件**：新建 `pages/search/search.js`, `pages/search/search.wxml`

**实现**：
1. 首页顶部搜索栏，点击跳转搜索页
2. 调用 `GET /recipes?keyword=xxx`
3. 结果列表点击跳转菜谱详情
4. 搜索历史：本地存储最近 10 条

**验收**：能搜索并查看结果

---

### Task 2.3：收藏功能

**目标**：收藏喜欢的菜谱

**文件**：菜谱详情页 + 新建 `pages/favorites/favorites.js`

**实现**：
1. 菜谱详情页添加★收藏按钮
2. 调用后端 `POST /recipes/:id/favorite`
3. 收藏列表页：调用 `GET /favorites`
4. 本地缓存收藏 ID 列表做离线展示

**验收**：能收藏、查看收藏列表

---

### Task 2.4：分享菜谱到微信群

**目标**：一键分享菜谱给家人

**文件**：菜谱详情页

**实现**：
1. 使用 `onShareAppMessage` 分享菜谱
2. 分享卡片显示：菜谱标题 + 缩略图 + "一鱼两吃"
3. 被分享者打开后跳转菜谱详情

**验收**：能分享菜谱到微信群

## Phase 3：体验优化

### Task 3.1：骨架屏加载

### Task 3.2：离线缓存

**目标**：无网时也能查看最近浏览的菜谱

**文件**：`utils/cache.js`

**实现**：
1. 菜谱详情页加载后存入 `wx.setStorageSync('cachedRecipes', ...)`
2. 首页加载失败时读取缓存展示
3. 标记"网络不可用"

---

### Task 3.3：菜谱预览图

**目标**：列表页显示缩略图

**文件**：`pages/recipe/recipe.wxml`

**实现**：
1. `GET /recipes` 返回数据包含 `imageUrl`
2. 列表渲染 `<image src="{{item.imageUrl}}">`
3. 无图时显示占位图

## 注意事项

1. 微信小程序需用真机调试，部分 API 模拟器不可用
2. 后端接口可能有 CORS 问题，需后端配置白名单
3. 用户隐私合规：`wx.getUserProfile` 需明确用途
4. 图片域名需在微信后台配置
