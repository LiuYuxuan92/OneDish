# 简家厨测试执行规范（必执行）

> 目的：保证每次测试前有计划、测试后有报告，可追溯、可复现、可审计。

## 1. 必备文档（缺一不可）

每次测试必须产出并提交以下两份文档：

1. **测试文档（Test Plan）**
   - 建议路径：`docs/testing/plans/YYYY-MM-DD-<主题>-test-plan.md`
2. **测试报告（Test Report）**
   - 建议路径：`docs/testing/reports/YYYY-MM-DD-<主题>-test-report.md`

---

## 2. 流程（标准）

1. 先写测试文档（计划、范围、用例、通过标准）
2. 再执行测试（按用例逐条记录）
3. 最后写测试报告（结果、问题、结论、风险）
4. 关联本次代码变更（commit / 文件清单）

---

## 3. 测试文档模板（最少字段）

- 测试主题
- 测试日期/执行人
- 代码范围（模块、文件、版本）
- 测试环境（前后端版本、端口、设备/浏览器）
- 测试范围（包含/不包含）
- 用例清单（用例ID、步骤、预期结果）
- 准入条件（开始测试前必须满足）
- 通过标准（Pass Criteria）
- 风险与回滚预案

---

## 4. 测试报告模板（最少字段）

- 测试主题
- 测试时间窗口
- 实际环境
- 执行摘要（总用例/通过/失败/阻塞）
- 用例执行结果明细
- 缺陷清单（严重级别、复现步骤、影响范围、当前状态）
- 偏差说明（与测试计划不一致项）
- 结论（是否允许进入下一阶段）
- 后续动作（责任人+截止时间）

---

## 5. 命名与归档规范

- 计划文档：`YYYY-MM-DD-<topic>-test-plan.md`
- 报告文档：`YYYY-MM-DD-<topic>-test-report.md`
- topic 用小写短横线，例如：`backend-build-smoke`

目录结构：

```text
docs/testing/
  plans/
  reports/
```

---

## 6. 执行纪律

- **没有测试文档，不开始测试。**
- **没有测试报告，不宣称完成。**
- 所有结论必须可复验（附命令、日志或截图证据）。
- 每轮测试必须记录“未覆盖项”。
