# CODE_REVIEW_CHECKLIST.md

## 1) 需求与范围
- [ ] 本次改动目标清晰
- [ ] 改动范围受控（无无关重构）
- [ ] 高风险改动是否已获批准

## 2) 代码质量
- [ ] 命名/结构清晰，可维护
- [ ] 无明显重复代码与死代码
- [ ] 错误处理完整（不吞错）
- [ ] 日志信息足够定位问题

## 3) 安全与数据
- [ ] 鉴权/权限链路未被破坏
- [ ] 敏感信息未硬编码
- [ ] 输入有校验，输出有边界
- [ ] 数据读写/迁移有回滚预案

## 4) 兼容与稳定
- [ ] 兼容现有接口契约
- [ ] 并发/幂等场景已考虑
- [ ] 配置变更影响面可控

## 5) 验证（必填）
- [ ] lint 通过
- [ ] test 通过（无测试需说明）
- [ ] build 通过
- [ ] 关键路径手测通过

## 6) 交付与回滚
- [ ] PR 信息完整（目标/改动/风险/验证）
- [ ] 回滚命令明确（`git revert <commit>`）
- [ ] 最终结论：可提交 / 不可提交
